# Cognitive ease review (Pass 5)

This pass focuses on reducing friction and mental load while adding new site utilities.

## What was adjusted to support cognitive ease

### 1) Progressive disclosure
- Tools still require a gate first (keeps complexity behind intent).
- The Tools page now links out to separate “Amenity Finder” and “Commute Analysis” pages (keeps the main Tools stack from becoming an endless scroll).

### 2) Consistent hierarchy
- All new pages follow the same structure:
  - Slim premium hero (eyebrow → strong headline → short explainer)
  - One core interactive panel
  - A short clarifier note (what it does / does not do)

### 3) Chunking and scannability
- Long paragraphs are limited in width.
- Metrics render in compact grids.
- Quick actions exist for the FAQ chatbot.

### 4) Cognitive load controls
- Clear labels and defaults for every tool.
- Inline error states for Amenity and Commute.
- Reduced-motion support (ticker stops animating when the user requests reduced motion).

### 5) Navigation clarity
- Footer includes direct links to Schedule, Amenity Finder, Commute Analysis, Privacy.
- Tools page includes quick links to Schedule/Amenities/Commute in the rail.

## Remaining opportunities
- Add a single “Utilities” section on Home and/or Tools to surface Schedule/Amenities/Commute without requiring scrolling.
- Add inline “examples” below the address fields (e.g. a GTA industrial address placeholder the user can click).
- Add saved-history for Amenity/Commute searches (localStorage) for repeat users.
